# 📺 live-programm — расписание ТВ-программ как TikTok

Робот сам находит расписание каналов в открытом интернете и публикует
статический сайт с вертикальной лентой в стиле TikTok: лайки, свайпы,
плеер `.m3u8`.

## Откуда берутся данные

Никаких выдуманных программ и заглушек вида «Утреннее вещание» — если для
канала не нашлось реального расписания, он публикуется без программы
(`no_data: true`), а не с фейком.

Робот (`scripts/epg-robot.js`, Node.js) на каждом запуске проходит четыре
шага:

1. **Публичные XMLTV-агрегаторы** (`epgshare01.online`, `iptv-org`-зеркала
   и т.д.) — общедоступные, без авторизации, обновляются сообществом
   ежедневно. Один запрос даёт расписание сразу для многих каналов.
2. **Настоящий поиск по интернету** для каналов, которым XMLTV не дал
   расписания: робот ищет `"{канал} программа передач сегодня"` через
   DuckDuckGo (HTML/Lite) с резервом на Bing — без API-ключей — и парсит
   найденные страницы (`scripts/htmlScheduleParser.js`, эвристика по
   парам «время + название», устойчивая к разной вёрстке сайтов).
3. **JSON API tv.yandex.ru** — последний резерв, если ни XMLTV, ни поиск
   не нашли расписание для канала. Это тоже настоящий, рабочий публичный
   API «Яндекс.Телепрограммы» (без авторизации), а не выдумка.
4. **Реальные m3u8-плейлисты** — привязка `stream_url` к каналам по
   названию из публичных плейлистов.

Параллельно на шагах 1 и 3 робот **находит новые каналы**, которых ещё
нет в реестре (`data/known-channels.json`), и добавляет их — список
каналов растёт от запуска к запуску без участия человека.

## Структура проекта

```
scripts/
  epg-robot.js         — главный оркестратор (шаги 1–4 выше)
  build-site.js         — собирает docs/ для GitHub Pages
  config.js              — источники, лимиты, тайминги
  seedChannels.js         — стартовый список каналов (без расписаний/моков)
  channelRegistry.js       — загрузка/сохранение растущего реестра каналов
  discovery.js               — сопоставление и добавление новых каналов
  htmlScheduleParser.js        — эвристический парсер найденных страниц
  sources/
    xmltv.js                    — публичные XMLTV-агрегаторы
    webSearch.js                  — DuckDuckGo/Bing без API-ключей
    yandexTv.js                     — резервный JSON API tv.yandex.ru
    playlists.js                      — парсинг m3u8-плейлистов
lib/
  http.js   — fetch с таймаутами/ретраями/gzip
  log.js     — логирование в stdout + data/robot.log
data/
  schedule.json   — итоговые данные (генерируется роботом)
  known-channels.json — растущий реестр каналов
index.html / embed.html / player.html — фронтенд (вертикальная лента)
```

## Запуск локально

```bash
npm install
npm run robot   # запускает поискового робота, пишет data/schedule.json
npm run build:site-only   # собирает docs/ из текущих data/
# или одной командой:
npm run build
```

Откройте `index.html` локально (или `docs/index.html` после сборки) —
данные уже встроены в `<script>window.__EPG_DATA__=...</script>`, либо
страница подтянет `data/schedule.json` через `fetch`.

## Автообновление

`.github/workflows/epg.yml` запускает робота каждые 3 часа на GitHub
Actions, коммитит обновлённые `data/*.json` и собранный `docs/`.

## Деплой

- **GitHub Pages** — из папки `docs/` после коммита workflow.
- **Vercel** — `vercel.json` уже настроен на корень репозитория.
